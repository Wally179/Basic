O Gargalo de Performance

A renderização inicial dos relatórios carregava mais de 25.000 nós no DOM simultaneamente. Isso causava o congelamento da Main Thread (block time > 4s) em laptops corporativos padrão.

O desafio era manter a funcionalidade de "Busca Global" e "Ordenação" no Front-end (para evitar round-trips ao servidor) sem travar a interface do usuário.

A Solução Técnica

Implementei uma estratégia de Virtualização (Windowing) customizada. Em vez de renderizar a lista inteira, o React mantém no DOM apenas os itens visíveis no viewport (+ um pequeno buffer).

Além disso, utilizei princípios **SOLID** para desacoplar a lógica de negócio da camada de visualização com **Styled Components**, garantindo manutenibilidade.

components/VirtualList.tsx
Performance Optimization 
// Hook customizado para gerenciar renderização virtual de 10k+ itens
export const useVirtualization = <T>(
  items: T[], 
  height: number
) => {
  const [scrollTop, setScrollTop] = useState(0);

  const visibleCount = Math.ceil(window.innerHeight / height);
  const startNode = Math.floor(scrollTop / height);
  
  // Renderiza apenas o necessário + buffer de segurança
  const visibleItems = useMemo(() => {
    return items.slice(
      startNode, 
      startNode + visibleCount + 5 // +5 Buffer items
    );
  }, [items, startNode, visibleCount]);

  return { visibleItems, startNode, totalHeight: items.length * height };
};
